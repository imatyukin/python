Документация к программе "Traffic Generator v.2.0"
Описание
"Traffic Generator v.2.0" - это приложение для генерации сетевого трафика с настраиваемыми параметрами.
Программа позволяет отправлять UDP и ICMP пакеты, а также настраивать скорость отправки, размер пакетов и
QoS (качество обслуживания). Программа предназначена для тестирования сетевой инфраструктуры,
проверки ее производительности и устойчивости к нагрузкам.

Основные функции
Генерация трафика: Отправка сетевых пакетов с заданными параметрами.
Поддержка протоколов: UDP и ICMP.
Настройка типа трафика: Unicast, Broadcast, Multicast.
Настройка IP адресов: Источник и назначения.
Настройка портов: Источник и назначения (только для UDP).
Настройка размера пакета.
Настройка количества потоков.
Настройка скорости трафика: kbps, mbps, pps или интервал между пакетами.
Настройка качества обслуживания (QoS): DSCP, IP Precedence, ECN.
Графическое отображение: Вывод графиков PPS (пакетов в секунду) и Mbps (мегабит в секунду).
Статистика: Отображение количества отправленных пакетов, общего объема отправленных байтов, средней скорости отправки в Mbps и PPS, количества ошибок.
Загрузка и сохранение конфигурации: Сохранение и загрузка параметров в/из YAML файла.

Интерфейс пользователя
Интерфейс приложения состоит из следующих элементов:
Заголовок: "Traffic Generator v.2.0".
Статусная строка: Выводит сообщения о текущем состоянии программы и ошибках.
Группа настроек "Settings":
Traffic Type: Выпадающий список для выбора типа трафика (unicast, broadcast, multicast).
Protocol: Выпадающий список для выбора протокола (UDP или ICMP).
Source IP: Текстовое поле для ввода IP-адреса источника.
Destination IP / Group IP: Текстовое поле для ввода IP-адреса назначения или группы. Зависит от выбранного типа трафика.
Source Port: Текстовое поле для ввода порта источника (только для UDP).
Destination Port: Текстовое поле для ввода порта назначения (только для UDP).
Packet Size: Текстовое поле для ввода размера пакета (в байтах).
Threads: Текстовое поле для ввода количества потоков.
DSCP (0-63): Текстовое поле для ввода значения DSCP (Differentiated Services Code Point) для QoS.
IP Precedence (0-7): Текстовое поле для ввода значения IP Precedence для QoS.
ECN (0-3): Текстовое поле для ввода значения ECN (Explicit Congestion Notification) для QoS.
Speed Mode: Выпадающий список для выбора режима скорости (kbps, mbps, pps, interval).
Speed Value: Текстовое поле для ввода значения скорости.
Кнопка "Start/Stop": Начинает или останавливает генерацию трафика.
Кнопка "Load Config": Загружает параметры из YAML файла.
Кнопка "Save Config": Сохраняет текущие параметры в YAML файл.
График: Отображает графики PPS и Mbps.
Поле статистики: Выводит статистику по отправленному трафику, включая количество отправленных пакетов, байтов, среднюю скорость и ошибки.

Работа с программой
Настройка параметров:
Выберите тип трафика (unicast, broadcast, multicast).
Выберите протокол (UDP или ICMP).
Введите IP-адрес источника.
Введите IP-адрес назначения (или групповой IP для multicast).
Введите порт источника (для UDP).
Введите порт назначения (для UDP).
Введите размер пакета в байтах.
Введите количество потоков (threads).
Задайте параметры QoS (DSCP, IP Precedence, ECN).
Выберите режим скорости и введите его значение.
Начало генерации трафика: Нажмите кнопку "Start".
Остановка генерации трафика: Нажмите кнопку "Stop".
Просмотр статистики: Наблюдайте за статистикой отправленного трафика и графиками.
Загрузка/Сохранение конфигурации: Нажмите кнопки "Load Config" или "Save Config" для загрузки или сохранения параметров.

Параметры трафика
Traffic Type:
unicast: Одноадресная передача, пакет отправляется одному получателю.
broadcast: Широковещательная передача, пакет отправляется всем устройствам в сети.
multicast: Многоадресная передача, пакет отправляется группе получателей.

Protocol:
UDP: Протокол без установления соединения, используется для быстрой передачи данных.
ICMP: Протокол контроля и диагностики сети, используется для отправки эхо-запросов (ping).

Source IP: IP-адрес источника трафика.
Destination IP / Group IP: IP-адрес назначения (или адрес группы для multicast).
Source Port: Порт источника (только для UDP).
Destination Port: Порт назначения (только для UDP).

Packet Size: Размер пакета в байтах. Максимальный размер пакета 65507 байт.

Threads: Количество потоков для генерации трафика (1-100). Увеличение количества потоков позволяет увеличить нагрузку.

DSCP (0-63): Значение DSCP для QoS, определяет приоритет пакета в сети.
IP Precedence (0-7): Значение IP Precedence для QoS, также влияет на приоритет пакета.
ECN (0-3): Значение ECN для QoS, используется для явного уведомления о перегрузке сети.

Speed Mode:
kbps: Скорость в килобитах в секунду.
mbps: Скорость в мегабитах в секунду.
pps: Скорость в пакетах в секунду.
interval: Интервал времени между отправкой пакетов (в секундах).

Загрузка и сохранение конфигурации
Конфигурация сохраняется и загружается в формате YAML.

Пример YAML файла:

protocol: UDP
traffic_type: unicast
source_address: 192.168.1.100
source_port: 12345
destination_address: 192.168.1.200
destination_port: 54321
packet_size: 512
threads: 4
speed_mode: mbps
speed_value: 10
qos:
  dscp: 46
  ip_precedence: 5
  ecn: 2
Use code with caution.

Технические требования
Операционная система: Windows, Linux, macOS.

Python 3.6 или выше.

Установленные библиотеки: PyQt5, matplotlib, pyyaml, qasync

Зависимости
Установите необходимые библиотеки перед запуском программы:

pip install PyQt5 matplotlib pyyaml qasync

Ограничения
Максимальный размер пакета: 65507 байт.
Максимальное количество потоков: 100.
Программа работает только с IPv4.
На Windows RAW-сокеты не позволяют задать ToS (DSCP) в IP-заголовке при создании собственного IP-заголовка.
